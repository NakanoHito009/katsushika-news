### 4-2. 操作エリア

#### 検索バー

| 項目 | 内容 |
|---|---|
| 種類 | テキスト入力欄 |
| 検索対象 | ニュースのタイトル・概要文 |
| 動作タイミング | 文字を入力するたびにリアルタイムで絞り込み |
| 大文字・小文字 | 区別しない |

#### カテゴリ絞り込みボタン

| ボタン名 | 対象カテゴリ | バッジ色 |
|---|---|---|
| 全て | すべて表示 | — |
| 区政 | 区の行政・予算・方針に関する情報 | 青 |
| 防災 | 災害・避難・ハザードマップなど | 赤 |
| イベント | 祭り・催し・施設利用など | オレンジ |
| 生活情報 | ごみ・道路工事・交通など | 緑 |
| 子育て | 保育・学校・子ども向けサービス | ピンク |
| 福祉 | 高齢者・障がい・健康など | 紫 |

- 選択中のボタンは背景が塗りつぶされて強調表示される
- 検索バーとの組み合わせ絞り込みに対応（例：「防災」カテゴリ内でキーワード検索）

#### 件数表示

- 現在表示中のニュース件数をリアルタイムで表示
- 例：`15件のニュース` → 絞り込み後 `3件のニュース`

### 4-3. ニュース一覧

#### レイアウト（画面幅による切り替え）

| 画面サイズ | 列数 | 対象デバイス |
|---|---|---|
| 861px 以上 | 3列 | PC・大型タブレット |
| 541px〜860px | 2列 | タブレット |
| 540px 以下 | 1列 | スマートフォン |

#### ニュースカードの構成

```
┌─────────────────────────┐
│ [カテゴリバッジ]         │  ← 色付きラベル（例：防災）
│                         │
│ タイトル                 │  ← 記事の見出し
│                         │
│ 概要文                   │  ← 記事の要約（最大80文字）
│                         │
│ 2026年04月21日  続きを読む→ │  ← 日付＋ボタン
└─────────────────────────┘
```

| 要素 | 仕様 |
|---|---|
| カテゴリバッジ | カード上部に色付きで表示。カテゴリごとに色が異なる |
| タイトル | 太字。複数行になる場合は折り返して全文表示 |
| 概要文 | 小さめのフォント。グレー文字 |
| 日付 | 「YYYY年MM月DD日」形式で表示 |
| 続きを読む | クリックで詳細モーダルを開く |
| ホバー効果 | カーソルを乗せると上に浮き上がるアニメーション |

#### 件数ゼロの場合

絞り込み結果が0件のとき、一覧の代わりに以下を表示：

```
📭
該当するニュースはありません
```

## 5. データ仕様

### ニュースデータの形式（news.json）

1件のニュースは以下の情報で構成されます。

| フィールド名 | 日本語 | 例 |
|---|---|---|
| id | 識別番号 | `"a1b2c3d4"` |
| title | タイトル | `"水元公園バーベキュー広場の予約受付を開始"` |
| category | カテゴリ名 | `"イベント"` |
| categorySlug | カテゴリ識別子 | `"event"` |
| date | 掲載日 | `"2026-04-08"` |
| summary | 概要文 | `"水元公園バーベキュー広場の..."` |
| url | 元記事のURL | `"https://www.city.katsushika.lg.jp/..."` |

### カテゴリの自動判定ロジック

葛飾区公式サイトのRSSには記事のカテゴリ情報が含まれていないため、**URLとタイトルのキーワードで自動判定**しています。

| 判定条件 | 割り当てカテゴリ |
|---|---|
| URLに `bousai`、タイトルに「防災」「避難」「地震」など | 防災 |
| URLに `kosodate`、タイトルに「子ども」「保育」「学校」など | 子育て |
| URLに `fukushi`、タイトルに「福祉」「介護」「高齢」など | 福祉 |
| URLに `kanko`、タイトルに「イベント」「祭り」「開催」など | イベント |
| URLに `gomi`、タイトルに「ごみ」「道路」「工事」など | 生活情報 |
| 上記に該当しない場合 | 区政（デフォルト） |

## 6. 自動更新の仕組み

```
毎日 午前6時（日本時間）
        ↓
GitHub Actions が自動起動
        ↓
fetch_news.py を実行
        ↓
葛飾区公式RSS（https://www.city.katsushika.lg.jp/news.rss）を取得
        ↓
最新30件を data/news.json に保存
        ↓
GitHub にコミット・保存
        ↓
サイトに自動反映
```

手動で今すぐ更新したい場合は、GitHub の Actions 画面から「Run workflow」を押すことで即時実行できます。

## 8. ローカル（自分のPC）で開く場合の注意

`index.html` をダブルクリックして開いた場合、ブラウザのセキュリティ制限により `news.json` を読み込めません。そのため、**サンプルデータ15件**が代わりに表示されます。

インターネット上の公開サイト（GitHub Pages）で開いた場合は、毎日更新される本物のデータが表示されます。
